📦 WAR 包部署时添加 Web 支持

  • 在 IntelliJ IDEA 中 添加 Web 支持，确保生成的 WAR 包包含正确的 Web 资源目录：
    1. 鼠标选中项目
    2. 双击 Shift 键打开搜索面板
    3. 输入 “add” 进行搜索
    4. 选择 Add Framework Support
    5. 勾选 Web Application

3. 前端页面返回与数据替换

❗ 页面返回问题

  • 直接返回完整 HTML 页面给前端可能较为麻烦，建议使用 静态 HTML 页面，后端通过模板引擎动态填充数据。

⚡ 解决方案

  1. 在 webapp 目录下编写 HTML 模板
  2. 后端仅返回 JSON 数据，前端 JavaScript 解析并渲染
  3. 使用 JSP、Freemarker 或 Thymeleaf 等模板引擎动态填充 HTML

4. 🔍 Tomcat 访问异常 Debug 流程

❓ 常见问题

当后端实现后，访问 Tomcat 时页面无法正常操作，可能出现以下几种情况：

  1. 404 Not Found（找不到资源）
  2. 405 Method Not Allowed（请求方法不匹配）
  3. 500 Internal Server Error（服务器内部错误）
  4. 前端请求无响应或数据未正确返回

🛠 排查步骤

✅ 检查 Tomcat 是否正常启动

  • 运行：

  • 检查终端日志是否有 启动成功 信息：

  • 如果 Tomcat 启动失败：

    • 检查端口是否被占用

    • 清理 Maven 构建

✅ 确保 Servlet 正常映射

  • 访问 http://localhost:8080/Login 是否能触发 LoginServlet
  • 若 404，确认 @WebServlet(urlPatterns = "/Login") 是否生效
  • 若 405，检查请求方式是否匹配（GET / POST）

✅ 调试 Servlet 参数获取

  • 在 doPost() 方法内添加：

  • 在 IDEA 中查看 Console 输出，确认参数是否正确传入

✅ 使用 Postman 测试 API

  • 直接发送 POST 请求到 http://localhost:8080/Login
  • 确保返回的数据正确无误

✅ 检查前端请求

  • 在浏览器按 F12 打开 开发者工具
  • 切换到 Network（网络）
  • 查找 请求路径 /Login 是否成功返回数据
  • 如果请求失败，检查：
    • 请求路径是否正确
    • 是否跨域（CORS）问题
    • 后端是否返回了 500 服务器错误

5. 🌟 总结

✅ 正确的目录结构

  • 确保 src/main/webapp 目录存在，并存放 HTML 资源
  • WEB-INF 目录存放 web.xml

✅ Tomcat 部署

  • 使用 mvn tomcat7:run 启动
  • 在 IDEA 中 添加 Web 支持

✅ 前后端交互

  • 后端提供 JSON API
  • 前端通过 JavaScript 解析并渲染

✅ Debug 流程

  1. 检查 Tomcat 是否正常启动
  2. 检查 Servlet 是否正确映射
  3. 在 Console 查看参数是否正确传入
  4. 使用 Postman 直接测试后端
  5. 使用浏览器 Network 面板排查请求
